**Stewardship invariants**

- `STEWARDSHIP` is the associative table for the **User ↔ CollabSite** many-to-many; `stewardship_role ∈ {Owner, Administrator, Reader, Writer}`.
- **Every CollabSite always has ≥ 1 `Owner`.** `Administrator` / `Reader` / `Writer` are each 0-to-many. In the ERD, `COLLAB_SITE ||--|{ STEWARDSHIP` enforces "≥ 1 steward"; the stricter "≥ 1 *Owner*" spans multiple rows, so it lives in app logic or a DB trigger (block deleting/demoting the last Owner), not a column `CHECK`.
- One role per (user, site): `UNIQUE(user_id, collab_site_id)`. If a user may hold several roles on one site, switch to `UNIQUE(user_id, collab_site_id, stewardship_role)`.

**Area shapes (one table, any shape)**

- `AREA_OF_STEWARDSHIP` replaces the old `PRINTABLE_MAP` + `PRINTABLE_MAP_AREA_CIRCLE` pair. Each row is **one shape** attached directly to a `CollabSite` — three concentric circles = three rows; a printable poster is just "render all of a site's areas," ordered by `sort_order`. One table (not one-per-shape) because every shape shares the same owner, styling, metadata, and spatial index — splitting by type would duplicate all of that.
- **`geom` is the universal shape column:** a generic `geometry(Geometry,4326)` (GeoDjango `GeometryField`) with a GIST index. It holds a Point, Polygon, MultiPolygon, etc., and **all** spatial queries (`ST_Contains`, `ST_DWithin`, `ST_Intersects`) run against it regardless of shape. `shape_type` only tells the editor/renderer how to treat the row.
- **Circles are stored parametrically** — `center` (Point) + `radius_m` — so you can render a crisp `L.circle(center, radius_m)` and let users drag the radius. Derive `geom` from them (`ST_Buffer(center::geography, radius_m)`) so the uniform queries still work. **Polygons** write their vertices straight into `geom` and leave `center`/`radius_m` null. So: exact circle math today, arbitrary polygons later, no schema change.
- A lone `PointField`/`geometry(POINT,4326)` can't represent a polygon, which is why the single shape column is `geom` (general geometry), with `center` kept as a Point only for the circle case.
- Open detail: `geom` as `geometry` (full function coverage incl. `ST_Contains`) vs `geography` (easy meter distances like `center`). Cheap to decide at migration time.
- Print/order concerns that may come later (page size, paid TIFF render) belong on their own table, not on the shape.

**Primary area (the site boundary)**

- `CollabSite` no longer stores geometry — center/boundary moved onto its areas. Every site has **exactly one primary** area (`is_primary = true`, seeded with `name = "Primary"`) that defines the outer stewardship boundary; other areas are expected to nest inside it. Keying on an `is_primary` flag rather than the display `name` keeps the structural role independent of a renamable label.
- Enforced rules (cross-row / value-specific → app logic or DB triggers, same class as the ≥1 Owner rule; the ERD can't express them):
  - **≥1 area per site:** `COLLAB_SITE ||--|{ AREA_OF_STEWARDSHIP` captures "≥1 area"; "≥1 *primary*" is the stricter part.
  - **Exactly one primary per site:** partial unique index — `CREATE UNIQUE INDEX ... ON area_of_stewardship (collab_site_id) WHERE is_primary`. (Caps it at one; guaranteeing *at least* one still needs app logic.)
  - **Primary can't be deleted alone:** `area.collab_site_id` is `ON DELETE CASCADE`, so deleting the *site* removes it; a guard blocks deleting or demoting the primary while its site still exists.
  - **Non-primary areas nest inside the primary (spatial):** every `is_primary = false` area must be geometrically contained by its site's primary — `ST_Contains(primary.geom, area.geom)` (swap in `ST_Covers` to let the edges touch). Cross-row *and* spatial, so it can't be a column `CHECK`; validate in app logic on create/update, or via a constraint trigger that looks up the site's primary `geom`. It's a spatial rule, not a foreign key, which is why `AREA_OF_STEWARDSHIP` has no self-reference in the diagram.
  - **Optional DB-level teeth:** add `COLLAB_SITE.primary_area_id FK → AREA_OF_STEWARDSHIP(id) NOT NULL ON DELETE RESTRICT` (a deferrable circular FK) to make "must have a primary, can't delete it" a Postgres guarantee. Left off the diagram to avoid circular-FK migration complexity unless you want it.
