## Why

The core entity of the app is a "Collab Site", the objects built from "collab_sites" rows.
In this story we'll implement the 6 screens that lead the admin user to a map centric layout showing a particular collab site map with Leaflet or other mapping tool.
Much of the day to day end user UI experience will revolve around the map created initially, so getting this flow and basic map features in place is key.

## How

This story introduces the earliest admin interface after initial bootstrapping

### Pre-UI usage to bootstrap an admin user with SSO setup

After the system has been bootstrapped from nothing, it cannot do anything until it has been configured with a SSO provider (Zitadel/SPNC-0001). Going to any URL route will return a 403 and message like "SSO not setup yet".

To bootstrap SSO, there will need to be a script or terraform project that can talk to hosted Zitadel instance. In SPNC-0001 Zitadel OAuth Client may have been setup manually, here we might consider doing this in a more automated way.

### GET /collab/new#set-goals

A form with 3 text fields and this prompt before it:

"Name 3 goals for the people living in this area that you will be collaborating with" . Anyone invited to the collab will see these prominently when using the site.

### GET /collab/new#write-about

A form with a text area that supports Markdown with this prompt:

"Write a short welcome message people will see before goals. This can be a welcome page, have some important contact info, or whatever you want your users to see when they login"

### GET /collab/new#preview-homepage

Renders the About, Goals, and Map like users will see when they login.

Something at the bottom or top header says "Your site will look like this to users"
