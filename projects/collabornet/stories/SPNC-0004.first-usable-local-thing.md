## Why

This is the **first code written for collabornet** and the foundation everything else
iterates from. The goal is the smallest thing that shows the *one experience that matters*:
a neighbor opens a page and sees their hyper-local commons as a **map** — a stewardship
area with resources (assets, skills, surplus) pinned on it.

[SPNC-0002](SPNC-0002.create-new-collab-site.md) builds an admin UI to *create* that seed
data. We invert that here: **hand-author the seed data in YAML and build the map-driven
end-user view directly.** That gets a real, demoable experience in a couple of hours — a
place to stand while figuring out which features actually matter before involving real
neighbors. Auth, RBAC, and multi-tenancy are deliberately out of scope (see decisions).

Scope ceiling: **buildable in a few hours to one day; runs from a laptop; demoable to one
person.** If a thing isn't needed to show "the map with pins," it doesn't belong here.

## How

Stack (per [build-vs-fork-recommendation.md](../research/build-vs-fork-recommendation.md)):
**GeoDjango + PostGIS**, server-rendered **Django template + Leaflet (CDN)**. No React
build step yet, no API layer, no auth.

### App layout (proposed)
```
projects/collabornet/app/
  manage.py
  collabornet/            # Django project (settings, urls, wsgi)
  commons/                # the one app: models, views, templates
    models.py             # CollabSite, Resource
    views.py              # home view -> renders the map page
    management/commands/seed.py   # wipes + loads YAML into the DB
    templates/commons/home.html   # About + Goals + Leaflet map
  seed/
    collab_site.yaml
    resources.yaml
  docker-compose.yml      # one service: postgis/postgis:15-3.4-alpine
```

### Models (lay the right foundation, even if v0 doesn't query spatially)
- `CollabSite`: `name`, `slug`, `center = PointField(geography=True, srid=4326)`,
  `radius_m = PositiveIntegerField`, `about_md = TextField`, `goals = JSONField` (list of 3).
- `Resource`: `collab_site = FK`, `kind = CharField(choices=[asset, skill_offer,
  skill_request, surplus])`, `title`, `description_md = TextField`,
  `location = PointField(geography=True, srid=4326)`.

`PointField` auto-creates the `geometry(POINT,4326)` column + GIST index, so the
`ST_DWithin`/radius-query foundation is in place for later without being used yet (v0 just
renders *all* of the single site's resources).

### Seed YAML (the "content", hand-authored)
`collab_site.yaml` — one site (e.g. a real MSP block you know):
```yaml
name: Pleasant Ave Commons
slug: pleasant-ave
center: { lat: 44.9418, lng: -93.2870 }
radius_m: 300
about_md: |
  Welcome to our block's shared resources...
goals:
  - Share tools so nobody buys what a neighbor already owns
  - Keep surplus food out of the trash
  - Know who can help with what
```
`resources.yaml` — a handful per kind:
```yaml
- kind: asset
  title: Cargo bike (borrow)
  description_md: Electric-assist, fits 2 kids. Text first.
  location: { lat: 44.9421, lng: -93.2861 }
- kind: skill_offer
  title: Bike repair
  description_md: Flats, brakes, tune-ups. Weekends.
  location: { lat: 44.9410, lng: -93.2885 }
- kind: surplus
  title: Tomato seedlings
  description_md: ~20 left, free, on the porch.
  location: { lat: 44.9425, lng: -93.2872 }
```

### `seed` management command
`python manage.py seed` — idempotent: wipes `CollabSite`/`Resource`, reads both YAML files,
creates rows. Re-running after editing YAML changes what the page shows (proves the page is
data-driven, not hardcoded).

### The page — `GET /` (single tenant: the one CollabSite)
Server-rendered template that shows, top to bottom, what a logged-in neighbor will
eventually see (mirrors [SPNC-0002](SPNC-0002.create-new-collab-site.md)#preview-homepage,
minus auth):
1. **About** (rendered from `about_md`).
2. **Goals** (the 3 goals, prominent).
3. **Map** (Leaflet, OSM raster tiles via CDN):
   - centered on `center`, with an `L.circle(center, {radius: radius_m})` drawing the
     stewardship area;
   - one marker per `Resource`, **visually distinguished by `kind`** (color/icon);
   - clicking a marker opens a popup with `title` + `description_md`.

v0 passes resources to the JS by inlining them as GeoJSON via Django's `json_script` (no
API endpoint needed yet). A `/resources.geojson` view + DRF-GIS is the natural first
iteration step, not part of this story.

## Notes & decisions
- 2026-06-16 — **GeoDjango over FastAPI** (research call): batteries-included spatial
  (PointField, GIST, GeoJSON, admin), Python, less boilerplate to first demo; bet is the
  framework stays current. Escape hatch if it ever stalls is documented in the research note.
- 2026-06-16 — **Single-tenant, no auth/RBAC.** Multi-tenant subdomains + Zitadel are a long
  way off; this story hardcodes the one site loaded from YAML.
- 2026-06-16 — **Django template + Leaflet (CDN), not React, for v0.** Fastest path to the
  map experience; React is a later iteration if interactivity demands it.
- 2026-06-16 — **PointField/PostGIS from day one** even though v0 renders all resources and
  runs no spatial query — keeps the `ST_DWithin` radius foundation in place. (SpatiaLite is a
  no-Docker fallback if standing up PostGIS locally proves annoying.)
- Cloud deploy is **out of scope** — that's [SPNC-0003](SPNC-0003.establish-cloud-deployment-mvp.md). This runs on the laptop only.
